Users CRUD API Documentation

by Caleb Aurand

GET /users

This route will return an array for each user item in our app. But only the county, first name, and last name are included.

GET /users:id

This route will return an array of one user object found by the user's id, with the fields of first_name, last_name, county, employed, and salary.

POST /users

This route can be used to create a user. The body of the request should include the first_name, last_name and county. The boolean if a user is employed, and their salary can be included, but is optional
{ "first_name": "Bob", "last_name": "Smith", "county": "Dallas", "employed": True, "salary": 25000 }

PUT /users:id

This route will update one user object's database values. It is updated by id. If the user is not found an error status code will be returned. The json body of the update request should include first_name, last_name and county. Optionally it can also include employed, and salary.
{ "first_name": "Bob", "last_name": "Smith", "county": "Dallas", "employed": True, "salary": 25000 }

DELETE /users:id

This route will delete one user found by the user's id, from the table of crudUsers. If the user's id is not found then an error status code will be returned.
